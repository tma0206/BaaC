## 1. Update（水やり）

### 1.1 前提条件

- 基質表面が乾いてから実行する（過剰な連続デプロイは根腐れリスク）。
- 鉢底から少量の排水が出るまでを「正常終了」の目安とする。

### 1.2 手順

1. **プリフライト:** 室温・風通しを確認。直射日光下での大量 Update は避ける。
2. **適用:** 土の表面からゆっくり注水し、一気に流し込まない（表面流走を防ぐ）。
3. **検証:** 数分後、葉のツヤと鉢の重みで含水状態を確認。
4. **ロールバック不可:** やりすぎた場合は風通しを上げて蒸散を促すのみ（「取り消し」はできない）。

### 1.3 スケジューリング（目安）

- 生育期: 基質乾燥に応じたイベント駆動。
- 休眠期: 頻度を下げ、Update サイズを小さく。

## 2. Refactoring（芽摘み）

### 2.1 目的

- 徒長枝の削除、密度の調整、樹形の可読性向上。
- マイクロ・リファクタリング: 小さな単位で繰り返し、**嗅覚フィードバック**（ヒノキ系の香り）を確認しながら進める。

### 2.2 手順

1. **差分確認:** 摘みたい芽・枝を特定（「このインターフェースは要らない」という判断）。
2. **適用:** 指でプチプチと摘む、または芽摘み鋏でカット。一度に大規模リライトしない。
3. **テスト:** 数日後、葉色と新芽の出方を観測。
4. **ドキュメント:** 大きめの剪定をした日付と理由をメモ（将来の自分へのコメント）。

### 2.3 注意

- 真柏は葉を剃りすぎると回復に時間がかかる。小さな PR を積み重ねるイメージで。
